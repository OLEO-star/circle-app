# GAS 再デプロイ手順（B+E セキュリティ強化 / 2026-06-02）

`code.gs` に **共有トークン検証（B）** と **日次自動生成上限（E）** を追加しました。
反映には Google Apps Script の **再デプロイが必要**です。以下を上から順に実施してください。

## 前提：デプロイ順序（重要）
1. フロント（`result/page.tsx`）は git push 済み → Cloudflare Pages が自動ビルド・公開中。
2. **先にフロントの公開完了を確認**してから GAS を再デプロイする。
   - 理由：GAS にトークン検証を入れると、トークンを送らない**古いフロント**からの送信は
     拒否される。新フロントが公開済みなら、その瞬間からトークン付きで送られるので安全。
   - 逆順（GAS先）だと、新フロント公開までの数分間の送信が拒否されてしまう。

## 手順

### 1. Cloudflare Pages の公開を確認
- ring-map.com の最新デプロイが「Success」になっているか Cloudflare ダッシュボードで確認。
- 念のため `ring-map.com/s` を開いて動作確認。

### 2. GAS エディタに新コードを貼り付け
1. https://script.google.com を開き、学部診断の集計プロジェクトを開く。
2. `code.gs` の中身を**全選択して削除**し、リポジトリの
   `circle-app/scripts/apps-script/code.gs` の**全文**を貼り付けて保存（⌘+S）。

### 3. 新しいデプロイを発行
1. 右上「デプロイ」→「デプロイを管理」。
2. 既存のウェブアプリデプロイの鉛筆（編集）アイコンをクリック。
3. 「バージョン」を「**新バージョン**」に変更。
4. 設定はそのまま（実行者：自分 / アクセス：**全員**）。
5. 「デプロイ」をクリック。
   - ※ URL は変わりません（編集デプロイなら `ANALYTICS_ENDPOINT` の差し替え不要）。
   - ※ もし「新しいデプロイ」で別URLを発行した場合は、`result/page.tsx` の
     `ANALYTICS_ENDPOINT` を新URLに更新して再 push が必要になります。**編集デプロイ推奨**。

### 4. 動作確認
1. `ring-map.com` で個人モードの診断を1回完走 → マスタ `diagnoses` に行が増えるか確認。
2. `ring-map.com/s` で学校モードを1回完走（事前登録校 or テスト校）→ 該当シートに記録されるか確認。
3. 何も記録されない場合 → トークン不一致の可能性。`code.gs` の `SECURITY.SHARED_TOKEN` と
   `result/page.tsx` の `POST_TOKEN` が完全一致しているか確認。

## 設定値（必要に応じて調整）
- `SECURITY.SHARED_TOKEN` … フロントの `POST_TOKEN` と一致必須。
  現在値：`b24ea11b1ee94af10fd8c48bba0215225ce8ae22`
- `SECURITY.DAILY_AUTOCREATE_CAP` … 1日に自動生成する学校別シートの上限（現在 5）。
  複数校が同日に正当導入するイベント時は一時的に引き上げる。

## 補足：このトークンの限界
公開Webアプリのため、トークンはブラウザに同梱され、本気で解析する人には抜き取られ得ます。
止められるのは「エンドポイント直叩き・スクリプト濫用」まで。完全な認証ではない点に留意。
より強固にするなら（エキスポ後）：reCAPTCHA、サーバ側プロキシ、レート制限などを検討。
